Proper Orthogonal Decomposition
(POD) in 2D

Extracting dominant deformation modes from high-fidelity 2D Snapshot Matrices.

1. The 2D Snapshot Matrix

In 2D Isogeometric Analysis, the solution vector $\mathbf{u}$ contains displacements for both $x$ and $y$ directions. For a mesh with $N_{cp}$ control points, the solution vector has a size of $2 \cdot N_{cp}$:

$$\mathbf{u} = [u_{x,1}, u_{y,1}, u_{x,2}, u_{y,2}, \dots, u_{x,N_{cp}}, u_{y,N_{cp}}]^T$$

We collect $N_s$ such solutions (snapshots) across different material parameters ($\mu$) into the **Snapshot Matrix** $\mathbf{S}$:

$$\mathbf{S} = [\mathbf{u}(\mu_1), \mathbf{u}(\mu_2), \dots, \mathbf{u}(\mu_{N_s})] \in \mathbb{R}^{(2N_{cp}) \times N_s}$$

2. Singular Value Decomposition (SVD)

POD identifies the most energetic "modes" of the system by performing SVD on the snapshot matrix. This factorizes $\mathbf{S}$ into three components:

$$\mathbf{S} = \mathbf{U} \mathbf{\Sigma} \mathbf{V}^T$$
  • $\mathbf{U}$ (Left Singular Vectors): Contains the **POD Modes**. Each column is a spatial basis function representing a fundamental way the structure deforms.
  • $\mathbf{\Sigma}$ (Singular Values): A diagonal matrix showing the "energy" or "importance" of each mode.
  • $\mathbf{V}$ (Right Singular Vectors): Describes how much of each mode is present in each original snapshot.

3. Energy Truncation & Efficiency

The core power of ROM comes from the fact that most physical systems are dominated by only a few modes. We truncate the basis to keep only $k$ modes, where $k \ll N_s$.

We choose $k$ by calculating the **Cumulative Relative Energy** ($\eta$):

$$\eta(k) = \frac{\sum_{i=1}^k \sigma_i^2}{\sum_{i=1}^{N_s} \sigma_i^2} > 0.9999$$

Typical Scree Plot: Exponential decay of singular values allows 99.9% energy capture with < 5 modes.

4. The Reduced Basis $\mathbf{\Phi}$

The final reduced basis matrix $\mathbf{\Phi}$ consists of the first $k$ columns of $\mathbf{U}$. This allows us to map the high-dimensional displacement $\mathbf{u}$ to a low-dimensional **Latent Space** $\mathbf{q}$:

$$\mathbf{u} \approx \mathbf{\Phi} \mathbf{q}$$

Solving for $\mathbf{q}$ (a vector of size $k$) is computationally negligible compared to solving for the full vector $\mathbf{u}$.
