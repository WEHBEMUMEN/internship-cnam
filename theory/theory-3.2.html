Phase 3.2 | Projection-Based ROM

Nonlinear Reduced Order
Modeling (NL-ROM)

Bridging the gap between high-fidelity nonlinear solvers and real-time interactive Digital Twins.

1. Snapshot Collection & POD

The first step in building a ROM is the Offline Training. We run the full nonlinear solver (FOM) for different load parameters and store the resulting displacement fields as "snapshots".

These snapshots form a matrix $\mathbf{S}$. We then apply **Proper Orthogonal Decomposition (POD)** via **Singular Value Decomposition (SVD)** to extract the most dominant "modes" of deformation:

SVD Extraction

$$\mathbf{S} = \mathbf{U} \mathbf{\Sigma} \mathbf{V}^T \implies \mathbf{\Phi} = \mathbf{U}[:, 1:k]$$

The reduced basis $\mathbf{\Phi}$ consists of $k$ vectors that capture $>99.9\%$ of the structural energy. Instead of solving for $N$ degrees of freedom, we now only solve for $k$ coordinates (where $k \ll N$).

2. Nonlinear Galerkin Projection

In nonlinear problems, the stiffness matrix $\mathbf{K}_T(\mathbf{u})$ and internal force $\mathbf{F}_{int}(\mathbf{u})$ depend on the displacement. To solve the reduced system, we project the full nonlinear residual and tangent stiffness into the reduced space:

Projected Operators

$$\mathbf{R}_{red} = \mathbf{\Phi}^T \mathbf{R}_{full}(\mathbf{\Phi} \mathbf{q})$$ $$\mathbf{K}_{red} = \mathbf{\Phi}^T \mathbf{K}_T(\mathbf{\Phi} \mathbf{q}) \mathbf{\Phi}$$

This allows us to solve a tiny $k \times k$ system in each Newton iteration. However, notice that we still need to compute the Full Order Residual $\mathbf{R}_{full}$ before projecting it. This leads to the "Galerkin Bottleneck".

3. The Galerkin Bottleneck

Even though the system we solve is small ($k \approx 5$), our engine currently evaluates the nonlinear internal forces over the entire mesh in every iteration. This means the computational cost still scales with the number of elements in the full-order model.

Future Milestone: Hyper-Reduction

To achieve true $100\times$ speedups, we must implement Hyper-Reduction (e.g., DEIM or ECSW). This will allow the solver to only evaluate physics at a tiny subset of "sampled" elements, bypassing the full mesh assembly entirely.
