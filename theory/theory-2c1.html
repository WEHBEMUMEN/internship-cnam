1. The Need to Sample Poisson's Ratio

In classical ROM benchmarks, researchers often sample simple variables like External Force ($\mathbf{F}$). However, modeling changes in **force** is a purely linear scalar operation ($\mathbf{u} = K^{-1}(\alpha \mathbf{F})$). It does not represent a true parametric digital twin.

To construct a rigorous ROM, we must sample the **material properties** themselves, forcing the stiffness matrix ($K$) to change continuously:

$$K(E, \nu) = \frac{E}{1-\nu^2} \begin{bmatrix} 1 & \nu & 0 \\ \nu & 1 & 0 \\ 0 & 0 & \frac{1-\nu}{2} \end{bmatrix}$$

By sampling both Young's Modulus ($E$) and Poisson's Ratio ($\nu$) using Latin Hypercube Sampling (LHS), we capture both "scale" and "coupling" deformations in our snapshot library.

2. The "Research-Ready" JSON

A major flaw in early ROM implementations is that saving just the numerical array of snapshots ($N_{dof} \times N_{samples}$) strips the data of its physical meaning. Without knowing *where* those degrees of freedom exist, the ROM cannot be visualized or verified.

Phase 2C.1 solves this by creating a **Self-Describing Topology Array**. We inject the full Isogeometric mathematical bounds directly into the `snapshots.json` export:

{
  "meta": {
    "geometry": { "R": 1.0, "L": 4.0 },
    "mesh": { "knotU": [...], "knotV": [...], "p": 3, "q": 3 }
  },
  "snapshots": [ ... ]
}

This ensures that any downstream script (like the SVD Lab) can instantly rebuild the physical 3D plate before reducing it.
