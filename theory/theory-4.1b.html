PHASE 4.1b THEORY

The Online Digital Twin Solver

Real-time reconstruction using the pre-trained ROM package.

1. Package Consumption

Phase 4.1b is the "execution" phase. It takes the JSON package exported from 4.1a, which contains:

  • Reduced Basis ($\mathbf{\Phi}$): The $N \times k$ matrix of POD modes.
  • Reduced Operators: $\mathbf{M}_r$ and $\mathbf{C}_r$ (the small $k \times k$ matrices).
  • ECSW Weights ($w$): The indices and values of the sampled elements.

2. Hyper-Reduced Force Evaluation

Instead of assembling the full stiffness matrix or internal force vector (which takes $O(N)$ time), the Online Solver only evaluates the forces on the **reduced integration domain** (RID):

$$\mathbf{F}_{int}^r(\mathbf{q}) \approx \sum_{e \in RID} w_e \mathbf{\Phi}_e^T \mathbf{f}_e^{int}(\mathbf{\Phi}_e \mathbf{q})$$

Where $\mathbf{\Phi}_e$ is the basis restricted to element $e$. Since $|RID| \ll |Elements|$, this evaluation is orders of magnitude faster, enabling the Digital Twin to respond in milliseconds.

3. Online Integration

The solver performs time integration in the reduced coordinate space $\mathbf{q}$. The system size is reduced from $1,000+$ DOFs to just $10$ or $20$, allowing the Newmark-β loop to run at high frequencies:

$$\mathbf{M}_r \ddot{\mathbf{q}} + \mathbf{C}_r \dot{\mathbf{q}} + \mathbf{F}_{int}^r(\mathbf{q}) = \mathbf{\Phi}^T \mathbf{F}_{ext}(t)$$

Full-State Visualization

Although the solver works in $\mathbf{q}$, we reconstruct the full displacement field $\mathbf{u} = \mathbf{\Phi}\mathbf{q}$ only for the WebGL visualizer. This ensures a high-fidelity visual experience with zero lag.
