Mathematics: Knot Vectors & Basis Functions

In the Phase 1.7 engine, the geometry is defined using Non-Uniform Rational B-Splines (NURBS). The domain is dictated by a Knot Vector ($U$), which divides the parametric space into elements.

The engine uses an Open Uniform Knot Vector. This means the ends of the knot vector are clamped with $p+1$ repeating values ($0$ at the start, $1$ at the end), forcing the curve to exactly interpolate the first and last control points.

$$U = [\underbrace{0, \dots, 0}_{p+1}, u_1, \dots, u_m, \underbrace{1, \dots, 1}_{p+1}]$$

The basis functions are computed recursively using the Cox-de Boor formula. This recursive approach builds higher-degree functions out of simpler linear blocks, maintaining strict local support and partition of unity.

$$N_{i,0}(\xi) = \begin{cases} 1 & \text{if } u_i \le \xi < u_{i+1} \\ 0 & \text{otherwise} \end{cases}$$ $$N_{i,p}(\xi) = \frac{\xi - u_i}{u_{i+p} - u_i} N_{i,p-1}(\xi) + \frac{u_{i+p+1} - \xi}{u_{i+p+1} - u_{i+1}} N_{i+1,p-1}(\xi)$$

To handle precise geometries like exact circular arcs, the engine extends B-Splines with weights ($w_i$), creating rational functions: $R_{i,p}(\xi) = \frac{N_{i,p}(\xi) w_i}{\sum N_{j,p}(\xi) w_j}$.

Variational Physics: Bending vs Axial Solvers

The physics engine features a Dual-Solver architecture, computing the stiffness matrix $\mathbf{K}$ using the Principle of Virtual Work. Depending on the mode selected, the structural behavior requires different derivatives of the shape functions.

Axial Stiffness (Truss / Bar)

In axial mode, elements only resist stretching or compression along their length. The strain energy depends entirely on the first derivative (gradient) of the basis functions.

$$K_{ij}^{axial} = \int_{0}^{L} EA \frac{d R_i}{d\xi} \frac{d R_j}{d\xi} d\xi$$

Flexural Bending (Euler-Bernoulli Beam)

In bending mode, elements resist curvature changes caused by transverse forces. Thus, the strain energy is proportional to the second derivative (curvature) of the basis functions.

$$K_{ij}^{bending} = \int_{0}^{L} EI \frac{d^2 R_i}{d\xi^2} \frac{d^2 R_j}{d\xi^2} d\xi$$

The matrix $\mathbf{K}$ is assembled by performing numerical integration over the parametric domain $\xi \in [0, 1]$ using Gauss Quadrature samples. This assembly handles the seamless mapping between parametric and physical space without needing individual element matrices.

Reduced Order Modeling (ROM) Detail

For a Digital Twin to be effective, it must synchronize with reality in real-time. If a high-fidelity finite element solution takes seconds to compute per frame, it fails to keep up with streaming sensor data. Isogeometric Analysis (IGA) solves this natively through rapid mathematical compression via Galerkin Projection.

1. The Modal Basis ($\mathbf{\Phi}$)

Instead of solving for the displacement of hundreds of individual control points, we assume the beam's final deflection is a combination of smooth, global shapes called modes. We generate a transformation matrix $\mathbf{\Phi}$ by sampling sine waves corresponding to the boundary conditions.

2. Galerkin Projection

Both the high-fidelity $[N \times N]$ Stiffness matrix $\mathbf{K}_{full}$ and the $[N \times 1]$ Force vector $\mathbf{F}_{full}$ are mathematically "squashed" down into the tiny $[m \times m]$ subspace defined by the modes.

$$\mathbf{K}_{red} = \mathbf{\Phi}^T \mathbf{K}_{full} \mathbf{\Phi}$$ $$\mathbf{F}_{red} = \mathbf{\Phi}^T \mathbf{F}_{full}$$

3. Solving the Reduced System

The engine solves the tiny reduced system $\mathbf{K}_{red} \mathbf{q}_r = \mathbf{F}_{red}$. The resulting vector $\mathbf{q}_r$ contains the modal amplitudes—values that simply dictate "how much" of each sine wave is needed to balance the applied physical force.

4. Fast Reconstruction

Finally, the full high-fidelity physical shape is instantly reconstructed for rendering directly in the browser by multiplying the abstract modal amplitudes back through the sine waves:

$$\mathbf{u}_{full} = \mathbf{\Phi} \mathbf{q}_r$$

Classical FEM shapes are often jagged ($C^0$ discontinuous), meaning they "fight" against smooth global sine waves during reduction, causing extreme errors. Since IGA bases are inherently $C^{p-k}$ smooth, they make the perfect fit for modal projection, allowing extreme 99% compression with nearly zero loss.

Force Positioning

Unlike traditional FEM where loads are often restricted to the discrete nodes between elements, IGA allows for precise loading at any continuous parametric coordinate $\xi \in [0, 1]$.

$$F_i = \int_{0}^{1} R_{i,p}(\xi) \, \delta(\xi - \xi_{load}) P \, d\xi = R_{i,p}(\xi_{load}) P$$

By sampling the basis functions $R_{i,p}(\xi_{load})$ at the exact point of application, the force vector $\mathbf{F}$ correctly distributes the point load across the surrounding control points smoothly.
