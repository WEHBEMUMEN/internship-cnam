The Limitation of Linear Statics

In the standard linear IGA solver from Phase 1.7, the governing equation is:

$$\mathbf{K}_{lin} \mathbf{u} = \mathbf{F}$$

The stiffness matrix $\mathbf{K}_{lin}$ is assembled once and never changes. This is valid for small deflections. However, when the applied load $\mathbf{F}$ is large enough that the beam deflects significantly, the geometry itself changes — the beam stretches, its arc length increases, and internal axial tension develops.

The linear solver is completely blind to these effects. If you push an immense force onto a cantilevered beam, the linear math will happily stretch the tip infinitely downward in a straight line. It will never curl, stiffen, or behave like a real physical structure because $\mathbf{K}_{lin}$ remains frozen at the undeformed configuration.

The Geometric Stiffness Matrix ($\mathbf{K}_{geom}$)

When a beam bends significantly, its centerline stretches. This stretching generates internal axial tension $N_{ax}$ that resists further bending — a phenomenon called stress stiffening. This is the von Kármán approximation for geometric non-linearity.

1 Compute the Induced Axial Force

The slope $v'(\xi)$ of the current deformed shape is evaluated using the IGA basis function derivatives. The induced axial tension is computed from the stretching energy:

$$N_{ax} = \frac{EA}{2L} \int_0^1 \left( v'(\xi) \right)^2 d\xi$$

2 Assemble $\mathbf{K}_{geom}$

The geometric stiffness matrix is assembled using the first derivatives of the IGA basis functions, scaled by the induced axial force $N_{ax}$:

$$(K_{geom})_{ij} = \int_0^1 N_{ax} \frac{dR_i}{d\xi} \frac{dR_j}{d\xi} d\xi$$

This matrix grows larger as the beam bends more — it's the mathematical representation of "the more you bend, the stiffer you get."

Newton-Raphson Iteration

The non-linear equilibrium equation becomes:

$$\mathbf{K}_T(\mathbf{u}) \, \Delta\mathbf{u} = \mathbf{R}(\mathbf{u})$$

where $\mathbf{K}_T = \mathbf{K}_{lin} + \mathbf{K}_{geom}(\mathbf{u})$ is the Tangent Stiffness and $\mathbf{R} = \mathbf{F}_{ext} - \mathbf{F}_{int}(\mathbf{u})$ is the Residual Force (the out-of-balance force). This is solved iteratively:

1 Assemble Tangent Stiffness

Build $\mathbf{K}_T(\mathbf{u})$ from the current deformed shape.

2 Compute Internal Force

$\mathbf{F}_{int}(\mathbf{u}) = \mathbf{K}_{lin} \mathbf{u} + \frac{1}{2} \mathbf{K}_{geom}(\mathbf{u}) \mathbf{u}$

3 Compute Residual

$\mathbf{R} = \mathbf{F}_{ext} - \mathbf{F}_{int}(\mathbf{u})$

4 Solve for Increment

$\Delta\mathbf{u} = \mathbf{K}_T^{-1} \mathbf{R}$

5 Update & Check Convergence

$\mathbf{u} \leftarrow \mathbf{u} + \Delta\mathbf{u}$. Repeat until $\|\Delta\mathbf{u}\| < \epsilon$.

Numerical Strategies Compared

The NL App allows you to compare different numerical strategies to solve the non-linear equation. Each has trade-offs between speed, stability, and accuracy per iteration:

Strategy Stiffness Update ($\mathbf{K}_{curr}$) Convergence Speed
Full Newton Updated every iteration Quadratic (Fastest)
Modified Newton Updated once per load step Linear (Slower)
Picard (Fixed Point) Secant / Lagged Stiffness Linear (Stable)
Incremental No iterations (Euler) Numerical Drift Error

Observation Goal

Notice that while Newton-Raphson converges in 3-5 iterations for most loads, the Incremental method results in a curve that "drifts" away from the true equilibrium because it never corrects the tangent error.

Non-Linear Reduced Order Model (NL-ROM)

In the linear ROM (Phase 1.7), Galerkin Projection maps the system into a modal subspace once. For non-linear problems, the Tangent Stiffness $\mathbf{K}_T$ changes every iteration — so the projection must be repeated at every Newton-Raphson step.

Step Full NL-IGA ($N \times N$) NL-ROM ($m \times m$)
Build Tangent $\mathbf{K}_T(\mathbf{u})$ at full DOF $\mathbf{K}_T(\mathbf{\Phi q}_r)$ then project
Project Not needed — already full $\mathbf{K}_{red-T} = \mathbf{\Phi}^T \mathbf{K}_T \mathbf{\Phi}$
Solve Gaussian Elimination ($O(N^3)$) $\mathbf{K}_{red-T} \Delta \mathbf{q}_r = \mathbf{R}_{red}$
Reconstruct Direct physical DOFs $\mathbf{u} = \mathbf{\Phi} \mathbf{q}_r$

The Key Insight

Even though we must rebuild $\mathbf{K}_T$ in full space each iteration (because the geometry-dependent $\mathbf{K}_{geom}$ requires the full displacement field), the actual solve operates on the tiny $[3 \times 3]$ reduced system. This means each iteration costs roughly $O(N^2)$ for projection but only $O(m^3)$ for the solve — a dramatic speedup when $m \ll N$.

NL-ROM Iteration Loop:

$$\mathbf{u}^{(k)} = \mathbf{\Phi} \mathbf{q}_r^{(k)}$$ $$\mathbf{K}_{red-T}^{(k)} = \mathbf{\Phi}^T \mathbf{K}_T(\mathbf{u}^{(k)}) \mathbf{\Phi}$$ $$\mathbf{R}_{red}^{(k)} = \mathbf{\Phi}^T \left( \mathbf{F}_{ext} - \mathbf{F}_{int}(\mathbf{u}^{(k)}) \right)$$ $$\mathbf{K}_{red-T}^{(k)} \Delta \mathbf{q}_r = \mathbf{R}_{red}^{(k)}$$ $$\mathbf{q}_r^{(k+1)} = \mathbf{q}_r^{(k)} + \Delta \mathbf{q}_r$$

Torque-to-Circle (Closed-Form Bypass)

Phase 1.7 also includes a closed-form analytical solution for the special case of a pure uniform bending moment $M$. When the moment is constant along the beam, the curvature $\kappa$ is also constant, producing a perfect circular arc:

$$\kappa = \frac{M}{EI} \implies r = \frac{1}{\kappa}$$

This bypasses the Newton-Raphson loop entirely and maps control points directly onto a circle. It serves as a visual validation tool — if the NL solver is correct, applying a uniform moment should produce a shape that closely matches this exact circular geometry.
