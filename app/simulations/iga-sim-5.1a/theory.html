Phase 5.1a: Geometric POD-Galerkin

Mathematical framework for real-time morphing of isogeometric digital twins using reduced-order modeling.

1. Isogeometric Analysis (IGA)

In Phase 5.1a, we utilize Isogeometric Analysis where the geometry is defined by NURBS (Non-Uniform Rational B-Splines). Unlike standard FEM, the basis functions used for the geometry are also used for the displacement field approximation.

u(u,v) = Σ Σ R_{i,j}(u,v) · d_{i,j}

Where R_{i,j} are the rational basis functions and d_{i,j} are the control point displacements. This ensures C¹ continuity across element boundaries, providing superior stress accuracy compared to C⁰ Lagrange polynomials.

2. Geometric Parameterization

The Digital Twin is defined by a parameter vector μ = [R, L₁, L₂]. Changing these values morphs the control point grid in real-time while maintaining the topological connectivity of the mesh.

Notch Radius (R)

Controls the internal circular boundary where stress concentration is highest.

Plate Width (L₁)

Scales the outer traction boundary, affecting the global stiffness.

Plate Height (L₂)

Adjusts the vertical symmetry plane, influencing the load path.

3. Proper Orthogonal Decomposition (POD)

To achieve real-time speeds, we reduce the problem size. We first harvest snapshots (full solutions) by varying the parameters in a 3D grid. These solutions are stored in a matrix S.

S = [u(μ₁), u(μ₂), ..., u(μₙ)] ∈ ℝ^{N × n}

We then perform a Singular Value Decomposition (SVD) on S:

S = U Σ Vᵀ

The Reduced Basis Φ is formed by the first k columns of U, which capture the most energy (variance) of the system.

4. Reduced Order Solve (Galerkin Projection)

Instead of solving the full system K u = f (size N), we project the equations into the reduced subspace defined by Φ.

u ≈ Φ · a

Substituting into the equilibrium equation and pre-multiplying by Φᵀ gives the Reduced System:

(Φᵀ K Φ) · a = Φᵀ f

The size of this system is k × k (e.g., 8 × 8), which is orders of magnitude smaller than the original 120 × 120 system, enabling microsecond response times.

5. Error Verification

We monitor the accuracy of the Digital Twin using the Relative L₂ Error. This metric compares the approximated solution to a background "True" solve performed in real-time for validation.

ε = || u_{FOM} - u_{ROM} ||₂ / || u_{FOM} ||₂

An error below 1% indicates high-fidelity geometric synchronization.
