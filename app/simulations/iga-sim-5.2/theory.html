Phase 5.1b: Geometric POD-Galerkin with ECSW

Detailed Mathematical Framework of Energy-Conserving Hyper-reduction

Objective: To solve parametric structural problems where the geometry $\Omega(\mathbf{\theta})$ is variable, using Energy-Conserving Sampling and Weighting (ECSW) to bypass the high-cost full mesh assembly during real-time online evaluations.

1. Exact Geometric Mapping (NURBS)

In IGA, the physical point $\mathbf{x}$ is a function of the parametric coordinate $\mathbf{\xi} = (\xi, \eta)$ and the shape parameter $\mathbf{\theta}$:

$$\mathbf{x}(\mathbf{\xi}; \mathbf{\theta}) = \frac{\sum_{i,j} \omega_{i,j} \mathbf{P}_{i,j}(\mathbf{\theta}) B_i^p(\xi) B_j^q(\eta)}{\sum_{i,j} \omega_{i,j} B_i^p(\xi) B_j^q(\eta)}$$

The **Jacobian Matrix** $\mathbf{J}$ represents the covariant basis of the transformation:

$$\mathbf{J} = \frac{\partial \mathbf{x}}{\partial \mathbf{\xi}} = \begin{bmatrix} \frac{\partial x}{\partial \xi} & \frac{\partial x}{\partial \eta} \\ \frac{\partial y}{\partial \xi} & \frac{\partial y}{\partial \eta} \end{bmatrix}$$

2. Strain-Displacement Operator in Variable Domains

The infinitesimal strain $\mathbf{\epsilon}$ is related to the control point displacements $\mathbf{d}$ via the $\mathbf{B}$-matrix. For a geometric twin, $\mathbf{B}$ is a function of $\mathbf{\theta}$ because it depends on the physical gradients of the shape functions:

$$\nabla_{\mathbf{x}} R = \mathbf{J}^{-1}(\mathbf{\theta}) \nabla_{\mathbf{\xi}} R$$ $$\mathbf{B}(\mathbf{\xi}; \mathbf{\theta}) = \begin{bmatrix} \frac{\partial R}{\partial x} & 0 \\ 0 & \frac{\partial R}{\partial y} \\ \frac{\partial R}{\partial y} & \frac{\partial R}{\partial x} \end{bmatrix} = \text{op}(\mathbf{J}^{-1} \nabla_{\mathbf{\xi}} R)$$

The differential area element also scales with the geometry:

$$dA = |\text{det}(\mathbf{J}(\mathbf{\xi}; \mathbf{\theta}))| d\xi d\eta$$

3. Proper Orthogonal Decomposition (POD) Algorithm

The Reduced Basis $\mathbf{\Phi}$ is computed using the method of snapshots. This algorithm compresses the high-dimensional displacement manifold into a low-dimensional subspace.

Algorithm 1: Offline POD Training
1. Input: Parameter space $\mathcal{D}$, Number of snapshots $N_s$.
2. Sampling: Generate $\Theta = \{\mathbf{\theta}_1, \dots, \mathbf{\theta}_{N_s}\}$ using Latin Hypercube Sampling (LHS).
3. Full Solves: For each $\mathbf{\theta}_i \in \Theta$:
   a. Morph control points: $\mathbf{P}_i = \mathcal{M}(\mathbf{P}_{ref}, \mathbf{\theta}_i)$.
   b. Assemble Stiffness: $\mathbf{K}(\mathbf{\theta}_i)$.
   c. Solve: $\mathbf{u}_i = \mathbf{K}(\mathbf{\theta}_i)^{-1} \mathbf{f}$.
4. Snapshot Matrix: Construct $\mathbf{S} = [\mathbf{u}_1, \dots, \mathbf{u}_{N_s}] \in \mathbb{R}^{N_{dof} \times N_s}$.
5. Economy SVD: Compute $\mathbf{S} = \mathbf{U}\mathbf{\Sigma}\mathbf{V}^T$.
6. Truncation: Retain $k$ columns of $\mathbf{U}$ s.t. $\sum_{j=1}^k \sigma_j^2 / \sum \sigma^2 > \text{tol}$.
7. Output: Reduced Basis $\mathbf{\Phi} = \mathbf{U}[:, 1:k]$.

4. Galerkin Projection and Hyper-reduction Bottleneck

The online solver approximates the solution as a linear combination of the POD modes: $\mathbf{u} \approx \mathbf{\Phi}\mathbf{a}$. Substituting this into the balance equation $\mathbf{K}(\mathbf{\theta})\mathbf{u} = \mathbf{f}$ and applying the standard Galerkin projection yields a reduced system:

$$\underbrace{(\mathbf{\Phi}^T \mathbf{K}(\mathbf{\theta}) \mathbf{\Phi})}_{\mathbf{k}_r(\mathbf{\theta})} \mathbf{a} = \underbrace{\mathbf{\Phi}^T \mathbf{f}}_{\mathbf{f}_r}$$

Although the projected system is small ($k \times k$), the reduced stiffness matrix $\mathbf{k}_r(\mathbf{\theta})$ is defined as the sum of all element stiffness matrix contributions projected onto the basis:

$$\mathbf{k}_r(\mathbf{\theta}) = \sum_{e=1}^{N_e} \mathbf{\Phi}^T \mathbf{K}_e(\mathbf{\theta}) \mathbf{\Phi}$$

The Bottleneck: Computing $\mathbf{k}_r(\mathbf{\theta})$ for a new geometric state $\mathbf{\theta}$ requires assembling all $N_e$ element stiffness matrices and projecting them. This operation scales with the full mesh size $N_e$, making real-time computation impossible on standard hardware.

5. Energy-Conserving Sampling and Weighting (ECSW)

ECSW bypasses the full mesh assembly by approximating the reduced stiffness matrix using a small, sparse subset of elements $E_{\text{sample}} \subset \{1, \dots, N_e\}$ along with a set of non-negative scaling weights $w_e > 0$:

$$\mathbf{k}_{r, \text{ECSW}}(\mathbf{\theta}) = \sum_{e \in E_{\text{sample}}} w_e \mathbf{\Phi}^T \mathbf{K}_e(\mathbf{\theta}) \mathbf{\Phi}$$

The Virtual Force System ($A w \approx b$)

To compute the weights $w_e$, we formulate a system of equations by matching the action of the stiffness matrix on the basis vectors across all training snapshots. For each snapshot $i = 1, \dots, N_{snaps}$ and basis vector $\mathbf{\phi}_j$ ($j = 1, \dots, k$):

$$\mathbf{A}_{e, i, j} = \mathbf{\Phi}^T \mathbf{K}_e(\mathbf{\theta}_i) \mathbf{\phi}_j \in \mathbb{R}^k$$ $$\mathbf{b}_{i, j} = \mathbf{\Phi}^T \mathbf{K}(\mathbf{\theta}_i) \mathbf{\phi}_j \in \mathbb{R}^k$$

By concatenating these vectors for all basis columns and training snapshots, we define a global virtual force matrix $\mathbf{A} \in \mathbb{R}^{D \times N_e}$ and a target vector $\mathbf{b} \in \mathbb{R}^D$ where $D = N_{snaps} \cdot k^2$:

$$\mathbf{A} = \begin{bmatrix} \mathbf{A}_1 & \mathbf{A}_2 & \dots & \mathbf{A}_{N_e} \end{bmatrix}, \quad \mathbf{b} = \sum_{e=1}^{N_e} \mathbf{A}_e$$

The optimal weights are found by solving a Non-Negative Least Squares (NNLS) optimization problem to promote sparsity:

$$\min_{\mathbf{w} \ge \mathbf{0}} \left\| \sum_{e=1}^{N_e} w_e \mathbf{A}_e - \mathbf{b} \right\|_2^2$$

6. Greedy OMP-NNLS Algorithm

The ECSW weights are solved using a greedy, Orthogonal Matching Pursuit (OMP) style active-set strategy to select the most energetic elements first:

Algorithm 2: Offline ECSW Hyper-reduction Training
1. Input: Target vectors $\mathbf{A}$, $\mathbf{b}$, Training Tolerance $\text{tol}$.
2. Initialization: Active Set $\mathcal{I} = \emptyset$, Residual $\mathbf{r} = \mathbf{b}$, weights $\mathbf{w} = \mathbf{0}$.
3. Loop: For $\text{iter} = 1$ to $\min(N_e, 200)$:
   a. Find element with max projection on residual: $e^* = \arg\max_{e \notin \mathcal{I}} \left( \mathbf{A}_e^T \mathbf{r} \right)$.
   b. If $e^*$ projection $\le 0$, break loop.
   c. Update Active Set: $\mathcal{I} = \mathcal{I} \cup \{e^*\}$.
   d. Solve Least Squares on active set: $\mathbf{w}_{\mathcal{I}} = \arg\min \|\mathbf{A}_{\mathcal{I}}\mathbf{w}_{\mathcal{I}} - \mathbf{b}\|_2^2$.
   e. Enforce non-negativity (clip): $\mathbf{w}_{\mathcal{I}} = \max(\mathbf{0}, \mathbf{w}_{\mathcal{I}})$.
   f. Update residual: $\mathbf{r} = \mathbf{b} - \sum_{e \in \mathcal{I}} w_e \mathbf{A}_e$.
   g. Check convergence: if $\|\mathbf{r}\|_2 / \|\mathbf{b}\|_2 < \text{tol}$, break.
4. Output: Sampled elements $E_{\text{sample}} = \mathcal{I}$, weights $\mathbf{w}_{E_{\text{sample}}}$.

7. Real-Time Online Hyper-reduced Solve

In the online stage, when a user morphs the geometry (changing shape parameter $\mathbf{\theta}_{\text{new}}$), the solver bypasses full mesh assembly entirely:

Algorithm 3: Online ECSW Hyper-reduced Solve
1. Input: Shape parameter $\mathbf{\theta}_{\text{new}}$, basis $\mathbf{\Phi}$, sampled elements $E_{\text{sample}}$, weights $\mathbf{w}$.
2. Local Stiffness Assembly:
   For each $e \in E_{\text{sample}}$:
      a. Compute local shape mapping $\mathbf{J}_e(\mathbf{\theta}_{\text{new}})$.
      b. Evaluate local strain operator $\mathbf{B}_e(\mathbf{\theta}_{\text{new}})$ and volume element $dA_e$.
      c. Assemble local element stiffness: $\mathbf{K}_e(\mathbf{\theta}_{\text{new}})$.
3. Weighted Projection: Assemble projected stiffness $\mathbf{k}_{r} = \sum_{e \in E_{\text{sample}}} w_e \mathbf{\Phi}^T \mathbf{K}_e(\mathbf{\theta}_{\text{new}}) \mathbf{\Phi}$.
4. Project Load Vector: $\mathbf{f}_r = \mathbf{\Phi}^T \mathbf{f}$.
5. Reduced Solve: Solve $k \times k$ system $\mathbf{k}_{r} \mathbf{a} = \mathbf{f}_r$.
6. Reconstruction: Reconstruct full displacement field: $\mathbf{u}_{ROM} = \mathbf{\Phi} \mathbf{a}$.
7. Output: Real-time displacement and stress visualizer.

8. Numerical Verification: Kirsch Benchmark

We use the relative $L_2$ error to certify the ROM and ECSW models against the Full Order Model (FOM):

$$\text{Error} = \frac{\sqrt{(\mathbf{u}_{f} - \mathbf{u}_{r})^T \mathbf{M} (\mathbf{u}_{f} - \mathbf{u}_{r})}}{\sqrt{\mathbf{u}_{f}^T \mathbf{M} \mathbf{u}_{f}}}$$

where $\mathbf{M}$ is the mass (or identity) matrix. For the notched plate, we target an error $< 1\%$ across the entire geometric sweep, while enjoying speedups of $10\text{x}$ to $100\text{x}$.
