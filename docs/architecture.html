Technical Architecture

The bridge between High-Fidelity IGA Solvers and Real-Time Digital Twin interfaces.

This architecture defines the transition from **1D Prototypes** (Phase 1) to **2D/3D Reduced Order Models** (Phase 2).

1. The Offline Forge

During the offline phase, we condense high-fidelity simulation data into a low-dimensional mathematical manifold. This is where the heavy lifting happens.

High-Fidelity Infrastructure

Using **Nutils** (Python-based IGA) to run nonlinear elastic simulations on multi-patch NURBS geometries.

Matrix Compression

Applying **POD (Proper Orthogonal Decomposition)** to find the optimal basis $\mathbf{\Phi}$ that captures 99.9% energy.

2. The Online Engine

The online phase is designed for millisecond responsiveness in the browser. We solve a projected system that mimics the complex physics at a fraction of the cost.

By solving for reduced coordinates $\hat{\mathbf{u}}$ of size $r \approx 10$, we bypass the need for inversions of size $2000 \times 2000$, resulting in a **100x to 1000x speedup**.

3. Deployment Strategy

Seamless Integration

- **Data**: Offline results serialized to `.json` and distributed via static CDNs.
- **Frontend**: A custom JavaScript/WebAssembly core projecting new parameter queries at **60 FPS**.
- **Visuals**: Three.js rendering of deformed NURBS surfaces natively mapped from the ROM output.
